REPL quirks
! Use the main editor (Shift+Enter) for multi-line code — the quick command box mangles indentation
! Comments work in the main editor only — # in the quick box causes IndentationError
! Patterns defined via REPL go to _pending_patterns first — they activate on the next bar, not instantly
! Always use the same function name as the slot (ch1, ch2…) when redefining — a new name creates a new pattern
! Multi-statement one-liners return OK not a value — split onto separate sends to inspect results
! print() outputs to the terminal, not the log — use bare expressions to see values here

import subsequence.constants.instruments.gm_drums as gm_drums
@composition.pattern(channel=10, length=4,
drum_note_map=gm_drums.GM_DRUM_MAP)
def ch10(p):
    p.seq("x ~ x ~", pitch="kick_1", velocity=100)
    p.seq("~ x ~ x", pitch="snare_1", velocity=90)
    p.seq("[x x] [x x] [x x] [x x]", pitch="hi_hat_closed", velocity=70)
